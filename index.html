Gem-AI Platform

Dashboard Interface

A Generative AI-powered analytics platform that democratizes data access by transforming natural language questions into executable SQL queries, interactive visualizations, and customizable dashboards.

Project Overview: Gem-AI bridges the gap between technical databases and non-technical stakeholders. Instead of relying on static reports, users can converse with their data directly. The application leverages Large Language Models (LLMs) to understand user intent, generate complex SQL queries, and instantly render the results as dynamic charts.

Key Features:

  • Text-to-SQL Engine: Translates natural language prompts (e.g., "Show monthly revenue") into optimized SQL queries, handling complex joins and providing "Explainable AI" verification.
  • Dynamic Visualization System: Intelligent auto-detection algorithms analyze query results to select the optimal visualization (KPIs, Line, Bar, etc.) and generate interactive filters.
  • Custom Dashboard Builder: A persistent, drag-and-drop workspace where users can pin insights, resize widgets, rename titles, and export full reports to PDF.

Technical Highlights:

  • Frontend: Built with vanilla JavaScript for maximum performance, using Chart.js for rendering and SortableJS for complex grid interactions.
  • Backend: Python/Flask API handling LLM orchestration and secure database execution.
  • State Management: LocalStorage-based persistence ensures user dashboards are saved instantly.

Tech Stack: Google Gemini Pro, LangChain, Python, Flask, SQL (PostgreSQL), HTML5, CSS3, JavaScript (ES6+), Chart.js, SortableJS, jsPDF.

Credibility Score AI

Credibility Analysis Visual

The Challenge: Addressing the need to reliably assess the credibility of information sources, especially in automated systems like RAG-based chatbots.

My Solution: Developed a proof-of-concept system to generate a credibility score for articles and references. This involved exploring Machine Learning, Rule-Based, and Hybrid approaches to analyze various textual and metadata features.

Key Features & My Role:

  • Researched and prototyped different scoring algorithms based on source authority, content analysis, and citation patterns.
  • Designed a framework for integrating this scoring mechanism into a Retrieval-Augmented Generation (RAG) pipeline.
  • Focused on identifying features indicative of bias, trustworthiness, and reference quality using NLP techniques.

Technologies Used: Python, Jupyter Notebooks, Natural Language Processing (NLP) libraries (e.g., NLTK, spaCy), RAG principles, Scikit-learn.
